Types of Cyber Security Technologies

Endpoint devices within many organisations have become necessary for employees to access office and remote work network resources. Ensuring efficient monitoring and securing of endpoint devices such as desktop computers, laptops, tablets, smartphones, and IoT devices is critical in any network infrastructure and cyber security strategy for protection against potential threats, data breaches and any unauthorised access whatsoever (Weber, 2021)

Endpoint Detection and Response solutions

There are a variety of methods used to safeguard endpoint devices. One such method is Endpoint Detection and Response (“EDR”) platforms. EDR is one of many cyber security tools used for detecting and analysing behaviours of endpoint devices by inspecting network traffic for suspicious patterns. One of EDR’s capabilities is to respond to detected threats such as malware, ransomware, and viruses by triggering an automatic response to the endpoint device to log off the end-user or send an alert to notify of a cyber threat at the endpoint level, often using automation (CORTEX, n.d.). A crucial part of any EDR security solution is its ability to proactively hunt for new threats that other Antivirus tools fail to discover (YourShortlist, 2022). This security function shortens the investigative time for an operations analyst to remediate any potential cyber threats. One weakness is its inability to install the agent-based EDR on unsupported operating systems and endpoint devices; however, agentless endpoint monitoring allows the tool to passively monitor network traffic that passes through the endpoint device, which serves as an alternate method (Elizabeth, 2022).

Firewall Protection

In addition to EDR security solutions, computer network stores sensitive and personal data that requires complete protection from malicious cyber-attacks on an organisation’s network servers and endpoint devices. One such solution is a perimeter-based firewall that operates at the OSI model’s layers 3 and 4 network level. Perimeter-based firewalls inspect and analyse network traffic entering and leaving an organisation’s secure environment from a less safe source. (Fortinet, n.d.) Firewalls are considered the first line of defence against cyber-attacks on a network and are an added layer of protection for endpoint devices and therefore are an essential part of an organisation’s cyber security strategy. Generally, there are two categories of Firewalls: hardware firewalls and host base firewalls. A hardware-based firewall is a security application that can be used as a perimeter-based firewall that filters, inspects and defends incoming and outgoing traffic from the internet destined for a private network and can block traffic based on predefined rules. (Hunter, 2021) A host-based firewall is a software application that runs on a computer system of endpoint devices connected to a network that protects the host device from malicious code and viruses that perimeter-based firewalls have not detected.

Peer Response - Steve

Steve, thanks for sharing an insight into packet-filtering firewalls. One can agree that companies must implement cyber security solutions at the network’s perimeter to separate the local area network from the internet of unwanted threats. As mentioned, the security of networked devices and the type of firewalls to implement are essential as the data and resources are protected.

Packet-filtering firewalls are known to be efficient, transparent, and affordable. They are also known to operate and work best where application security is not a concern. The focus is to block or accept data packet transmission entering and leaving the network based on protocols, and user define rules, ports and IP addresses (UNext Editorial Team Content Writer, 2020).

Depending on the user-defined packet-filtering rules, Static, Dynamic and Stateful are three types of rules assigned. Static packet filtering firewall rules set port connections between networks in the open or closed form by manual configuration. Dynamic packet filtering firewall rules open ports when a connection is established and closes when no link is available; otherwise, ports can remain closed by default. Stateful firewalls can identify UDP and TCP streams of traffic entering between points and distinguish between malicious and legitimate traffic flow (SecurityX, n.d.). One drawback with static and dynamic packet filtering firewall rules is their inability to offer address spoofing protection from hackers. Stateful packet filtering rules do eliminate some risk of address spoofing techniques.

After a small amount of research on air-gap techniques, the understanding is that Hackers use USB drives to breach computer networks by installing malware. To avoid these attacks, access to USB drives on air-gap systems should be disabled and only enabled for internal use by cyber security teams as a policy (Gillis, 2022).

Peer Response - Stella

Stella, thank you for sharing this fascinating topic on cyber security technologies that protect a computer network from cyber threats. Due to security threats, cyber-attacks, traffic flooding, cost and technical requirements, more extensive private networks can take considerable work to implement new technologies, such as firewalls and Intrusion Detection Systems outlined in this discussion.

To elaborate on these two technologies and the protection they provide to organisations from cyber-attacks, firewalls can be hardware or software and are designed to block malicious traffic from entering a private network, mainly from the internet, by blocking other networks, ports, and applications based on a set of security rules applied while allowing legitimate traffic to access network resources. The primary function of a firewall is to protect against malicious traffic and not malicious code (i.e., malware) that might not be blocked if accidentally installed, creating a backdoor for a threat actor to penetrate a computer system (Cybersecurity & Infrastructure Security Agency, 2019).

Using an Intrusion Detection System on a network helps the firewall monitor and analyse network traffic for anomaly behaviour. It can take considerable time to distinguish between regular activity and malicious traffic, including sensitive data like IP addresses and personal data; this can also contribute to false positives (Fedorchenko, et al., 2022).

When an IDS system detects suspicious traffic, it immediately sends you an alert in the early stages of the intrusion. Still, it does not prevent the attack from happening but allows an operations analysis to respond to the attack to mitigate them. Therefore, implementing intrusion prevention capabilities to extend the IDS system would be beneficial to securing the network end to end (Tek-Tools, 2020).

Although Circuit-level gateway firewalls ensure the request between handshakes is confirmed legitimate, one disadvantage is that the individual packets are not filtered once a session is authorised. (Geeks for Geeks, 2022)

Summary Post

The writer and commenters previously discussed the topic of Endpoint devices and Local Area Networks (“LAN”), which were ultimately found to be susceptible to cyber-attacks in an interconnected network if not adequately protected from unauthorised access and malicious code with the assistance of cybersecurity technologies. It was determined that using combined cybersecurity technologies versus relying on a single technology for protection was the most reliable method for comprehensive protection against potential threats. The panel were in unanimous agreement that endpoint protection is crucial to safeguarding an organisation’s technological assets.

The panel discussed two (2) primary forms of cybersecurity technologies utilised for the identification of potential threats and defence against active attacks. These technologies were Endpoint Detection and Response (“EDR”) - used to protect against malicious code - such as malware, ransomware and viruses, which a user could erroneously execute. The writer first noted that EDR is used for detecting and analysing behaviours of endpoint devices by inspecting network traffic for suspicious patterns. Commenter 1 added that extensive research was ongoing on the effectiveness of EDR with machine learning ability; however, it was noted that this newer method has limitations and may suffer performance issues for various reasons. One of the core benefits of EDR, as highlighted by Commenter 2, was that EDR could provide real-time information about incidents and disconnect a device from the network should it identify malicious activity.

Secondly, Perimeter-based firewalls were assessed, noting that these are the first line of defence used to strengthen a network from cyberattacks. Commenter 2 noted that Perimeter firewalls use various methods to respond to a network intrusion; the method or methods applied are entirely determined by the type of intrusion detected at any given time. For example, a perimeter firewall can block a banned website (‘static packet filtering’) or only allow return traffic based on traffic sent (‘stateful inspection’) (Kumar & Majumder, 2020). Commenter 1 accurately raised the issue of a perimeter firewall’s inability to detect threats or malicious activity that has already bypassed the firewall ( (Abie, 2000) – this highlights the need for multiple layers of cybersecurity to ensure comprehensive network coverage.

The panel ultimately determined that EDR and Perimeter firewalls are two effective and proven cybersecurity technologies used to protect a network by deterring, identifying and mitigating malicious threats as appropriate. Utilising these technologies in a combined effort to enhance a network’s security can be a defining factor of a successful security framework.
